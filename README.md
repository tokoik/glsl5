# glsl4 - 第４回 GLSL によるバンプマッピング サンプルプログラム

## 1. 概要

このプログラムは、OpenGL と GLSL (OpenGL Shading Language) を用いて「バンプマッピング (Bump Mapping)」を実装するための、学生向けのサンプルプログラムです。本プログラムは、以下のブログ記事の解説に沿って学習を進めるための雛形として提供されています。

- [第４回 GLSL によるバンプマッピング](https://tokoik.github.io/blog/glsl%20%E5%85%A5%E9%96%80/2005/10/14/glsl.html)

このプログラムは、以前の「球にバンプマッピング」するサンプルプログラムを GLSL を使って書き直したものであり、接空間（Tangent Space）におけるライティング計算をシェーダ内で行うことで、より効率的で柔軟なバンプマッピングを実現しています。

## 2. ビルド方法

このプログラムは [CMake](https://cmake.org/) を用いてビルド環境を整備します。各OSとも、ソースコードが置かれているディレクトリにターミナル（またはコマンドプロンプト）で移動してから、以下の手順を実行してください。なお、プログラムをビルドするためのバイナリディレクトリは、バージョン管理ファイル（.gitignore）の設定に合わせて **build** という名前にします。

### 2.1 Windows (Visual Studio 2022 の場合)

1. コマンドプロンプトまたは PowerShell を開き、このプロジェクトのディレクトリに移動します。
2. 以下のコマンドを実行してビルドディレクトリを作成し、CMake で構成を行います。

   ```bat
   mkdir build
   cd build
   cmake .. -G "Visual Studio 17 2022"
   ```

3. 生成された build フォルダ内の glsl4.sln を Visual Studio で開きます。
4. ソリューションエクスプローラーで **glsl4** プロジェクトを右クリックし、「スタートアップ プロジェクトに設定」を選択します。
5. 「ローカル Windows デバッガー」をクリックするか、F5 キーを押してビルドおよび実行します。

### 2.2 macOS (Xcode の場合)

1. ターミナルを開き、このプロジェクトのディレクトリに移動します。
2. 以下のコマンドを実行してビルドディレクトリを作成し、Xcode 用のプロジェクトを生成します。

   ```sh
   mkdir build
   cd build
   cmake .. -G Xcode
   ```

3. 生成された build/glsl4.xcodeproj を Xcode で開きます。
4. 左上のスキーム選択（再生ボタンの横）が **glsl4** になっていることを確認します。
5. 「Run」ボタン（再生ボタン）をクリックするか、Command + R を押してビルドおよび実行します。

### 2.3 Ubuntu Linux

1. ターミナルを開き、このプロジェクトのディレクトリに移動します。
2. 必要なパッケージ（freeglut3-dev など）がインストールされていることを確認し、以下のコマンドでビルドします。

   ```sh
   mkdir build
   cd build
   cmake ..
   make
   ```

## 3. 使い方

### 3.1 プログラムの起動方法

各OSとも、ビルド後に生成されるバイナリディレクトリ (build) やそのサブフォルダから起動します。（※ CMake の設定により、Windows や Xcode では Debug などのフォルダ下に実行ファイルが置かれることがあります）

- **Windows**

  Visual Studio 上で「ローカル Windows デバッガー」をクリックして実行するか、またはコマンドプロンプトから以下のコマンドで起動します。

  ```cmd
  cd build\Debug
  glsl4.exe
  ```

- **macOS**

  Xcode 上で左上の「Run（再生ボタン）」をクリックするのが楽です。これにより glsl4.app アプリケーションバンドルとして自動的に実行されます。アプリケーションバンドルを直接起動するなら、Finder から build/Debug/glsl4.app をダブルクリックするか、ターミナルから open build/Debug/glsl4.app を実行します (この場合はエラーメッセージ等が表示されません)。

- **Ubuntu Linux**

  ターミナルから以下のコマンドで実行ファイル（バイナリ）を直接起動します。

  ```sh
  cd build
  ./glsl4
  ```

### 3.2 操作方法

- **マウスの左ボタンでドラッグ**

  画面内のオブジェクト（球）を３次元的に回転させることができます。

- **キーボードの q, Q または ESC キー**

  プログラムを終了します。

## 4. 解説

このプログラムは、OpenGL と GLSL を用いて **バンプマッピング** を実装したものです。法線マップ（Normal Map）を使用してオブジェクト表面の凹凸を擬似的に表現します。接空間（Tangent Space）を用いたライティング計算をバーテックスシェーダおよびフラグメントシェーダ内で実行します。

### 4.1 アプリケーション側の処理 (main.cpp, sphere.cpp)

アプリケーション側では、シェーダプログラムの準備と、接空間の基底となる **接線ベクトル（Tangent Vector）** をシェーダに渡す処理を行います。

1. **シェーダプログラムの読み込みと変数の取得 (`init()` 関数)**
   - bump.vert（バーテックスシェーダ）と bump.frag（フラグメントシェーダ）を読み込みます。
   - `glGetAttribLocation(` `gl2Program` `,` `"tangent"` `)` を用いて、頂点の接線ベクトルを渡すための attribute 変数の場所を `tangentLoc` に取得します。
   - `glGetUniformLocation(` `gl2Program` `,` `"color"` `)` で、法線マップのサンプラを渡す uniform 変数の場所を `colorLoc` に取得します。
   - 高さマップ画像 dotbump.raw から `makeNormalMap()` 関数を用いて法線マップを生成し、テクスチャとして割り当てます。

2. **接線ベクトルの算出と設定 (sphere.cpp)**
   - 球を描画する際に、各頂点の法線ベクトル (`normal`) から接線ベクトル (`tangent`) を算出します（`setTangent()` 関数）。ここでは、球の極（y軸）方向を基準に計算しています。
   - `glVertexAttrib3dv(` `tangentLoc` `,` `tangent` `)` を呼び出すことで、算出した接線ベクトルを attribute 変数としてバーテックスシェーダに渡します。

### 4.2 バーテックスシェーダ (bump.vert)

バーテックスシェーダでは、頂点単位で**接空間への変換**と、光線・視線ベクトルの計算を行います。

1. **接空間の基底ベクトルの計算**
   - 視点座標系の法線ベクトル $\mathbf{n}$ (`normal`) を計算します。
   - アプリケーションから渡された接線ベクトル `tangent` を視点座標系に変換して正規化し、$\mathbf{t}$ とします。
   - 法線ベクトルと接線ベクトルの外積から、従接線ベクトル（Binormal / Bitangent）$\mathbf{b}$ を求めます（`b = cross(n, t)`）。

2. **ベクトルの接空間への変換**
   - 視点座標系で求めた光線ベクトル `light` と視線ベクトル `view` を、$\mathbf{t}$, $\mathbf{b}$, $\mathbf{n}$ を用いて接空間に変換し、それぞれ `tlight`, `tview` としてフラグメントシェーダに送ります。
   - これにより、フラグメントシェーダ側では法線マップから取得した法線（接空間基準）をそのまま使ってライティング計算ができるようになります。

```glsl
// 接空間における光線ベクトルの計算例
tlight.x = dot(t, light);
tlight.y = dot(b, light);
tlight.z = dot(n, light);
```

### 4.3 フラグメントシェーダ (bump.frag)

フラグメントシェーダでは、法線マップから取得した法線ベクトルを用いて、画素（フラグメント）単位の陰影（ライティング）計算を行います。

1. **法線マップからの法線ベクトルの取得**
   - `texture2DProj()` を用いて法線マップから色（RGB）を取得します。
   - 色の値は `0.0` 〜 `1.0` の範囲に収められているため、`color.rgb * 2.0 - 1.0` の計算によって `-1.0` 〜 `1.0` の範囲を持つ接空間の法線ベクトル `fnormal` に復元します。

2. **接空間でのライティング計算**
   - バーテックスシェーダから受け取った接空間の光線ベクトル `tlight` と視線ベクトル `tview` を使って陰影計算を行います。
   - 補間されたベクトル `tlight` を再正規化して `flight` を求めます。
   - 法線ベクトル `fnormal` と光線ベクトル `flight` の内積から拡散反射率 `diffuse = max(dot(fnormal, flight), 0.0)` を計算します。
   - 中間ベクトル `fhalfway = normalize(tlight + tview)` を求め、これと法線ベクトルから鏡面反射率 `specular` を計算します（Blinn-Phong モデル）。
   - 最終的に得られた反射率と光源の色を用いて最終的なフラグメントの色 `gl_FragColor` を出力します。
