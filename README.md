# glsl5 - 第５回 環境マップバンプマッピング サンプルプログラム

## 1. 概要

このプログラムは、OpenGL と GLSL (OpenGL Shading Language) を用いて「バンプマッピング (Bump Mapping)」を実装するための、学生向けのサンプルプログラムです。本プログラムは、以下のブログ記事の解説に沿って学習を進めるための雛形として提供されています。

- [第５回 環境マップバンプマッピング](https://tokoik.github.io/blog/glsl%20%E5%85%A5%E9%96%80/2005/10/15/glsl.html)

このプログラムは、前回の「[第４回 GLSL によるバンプマッピング](https://tokoik.github.io/blog/glsl%20%E5%85%A5%E9%96%80/2005/10/14/glsl.html)」のサンプルプログラムをもとに、バンプマッピングに加えて環境マッピングを施す、いわゆる**環境マップバンプマッピング** (Environment-Mapped Bump Mapping, EMBM) の実装を行おうとするものです。

## 2. ビルド方法

このプログラムは [CMake](https://cmake.org/) を用いてビルド環境を整備します。各OSとも、ソースコードが置かれているディレクトリにターミナル（またはコマンドプロンプト）で移動してから、以下の手順を実行してください。なお、プログラムをビルドするためのバイナリディレクトリは、バージョン管理ファイル（.gitignore）の設定に合わせて **build** という名前にします。

### 2.1 Windows (Visual Studio 2022 の場合)

1. コマンドプロンプトまたは PowerShell を開き、このプロジェクトのディレクトリに移動します。
2. 以下のコマンドを実行してビルドディレクトリを作成し、CMake で構成を行います。

   ```bat
   mkdir build
   cd build
   cmake .. -G "Visual Studio 17 2022"
   ```

3. 生成された build フォルダ内の glsl5.sln を Visual Studio で開きます。
4. ソリューションエクスプローラーで **glsl5** プロジェクトを右クリックし、「スタートアップ プロジェクトに設定」を選択します。
5. 「ローカル Windows デバッガー」をクリックするか、F5 キーを押してビルドおよび実行します。

### 2.2 macOS (Xcode の場合)

1. ターミナルを開き、このプロジェクトのディレクトリに移動します。
2. 以下のコマンドを実行してビルドディレクトリを作成し、Xcode 用のプロジェクトを生成します。

   ```sh
   mkdir build
   cd build
   cmake .. -G Xcode
   ```

3. 生成された build/glsl5.xcodeproj を Xcode で開きます。
4. 左上のスキーム選択（再生ボタンの横）が **glsl5** になっていることを確認します。
5. 「Run」ボタン（再生ボタン）をクリックするか、Command + R を押してビルドおよび実行します。

### 2.3 Ubuntu Linux

1. ターミナルを開き、このプロジェクトのディレクトリに移動します。
2. 必要なパッケージ（freeglut3-dev など）がインストールされていることを確認し、以下のコマンドでビルドします。

   ```sh
   mkdir build
   cd build
   cmake ..
   make
   ```

## 3. 使い方

### 3.1 プログラムの起動方法

各OSとも、ビルド後に生成されるバイナリディレクトリ (build) やそのサブフォルダから起動します。（※ CMake の設定により、Windows や Xcode では Debug などのフォルダ下に実行ファイルが置かれることがあります）

- **Windows**

  Visual Studio 上で「ローカル Windows デバッガー」をクリックして実行するか、またはコマンドプロンプトから以下のコマンドで起動します。

  ```cmd
  cd build\Debug
  glsl5.exe
  ```

- **macOS**

  Xcode 上で左上の「Run（再生ボタン）」をクリックするのが楽です。これにより glsl5.app アプリケーションバンドルとして自動的に実行されます。アプリケーションバンドルを直接起動するなら、Finder から build/Debug/glsl5.app をダブルクリックするか、ターミナルから open build/Debug/glsl5.app を実行します (この場合はエラーメッセージ等が表示されません)。

- **Ubuntu Linux**

  ターミナルから以下のコマンドで実行ファイル（バイナリ）を直接起動します。

  ```sh
  cd build
  ./glsl5
  ```

### 3.2 操作方法

- **マウスの左ボタンでドラッグ**

  画面内のオブジェクト（球）を３次元的に回転させることができます。

- **キーボードの q, Q または ESC キー**

  プログラムを終了します。

## 4. 解説

このプログラムは、OpenGL と GLSL を用いて **環境マップバンプマッピング (EMBM)** を実装したものです。法線マップ（Normal Map）を使用してオブジェクト表面の凹凸を表現するとともに、その凹凸を反映した法線ベクトルに基づいてキューブマップをサンプリングし、周囲の環境の映り込みを表現します。

### 4.1 アプリケーション側の処理 (main.cpp, sphere.cpp)

アプリケーション側では、シェーダプログラムの準備、接空間の基底となる **接線ベクトル（Tangent Vector）** をシェーダに渡す処理、およびテクスチャ（法線マップと環境マップ）の設定を行います。

1. **シェーダプログラムの読み込みと変数の取得 (`init()` 関数)**
   - embm.vert（バーテックスシェーダ）と embm.frag（フラグメントシェーダ）を読み込みます。
   - `glGetAttribLocation(gl2Program, "tangent")` により、頂点の接線ベクトルを渡すための attribute 変数の場所を `tangentLoc` に取得します。
   - `glGetUniformLocation(gl2Program, "color")` で、法線マップのサンプラを渡す uniform 変数の場所を `colorLoc` に取得します。
   - `glGetUniformLocation(gl2Program, "environment")` で、環境マップのサンプラを渡す uniform 変数の場所を `envronmentLoc` に取得します。

2. **法線マップの設定 (`init()` 関数)**
   - 高さマップ画像 dotbump.raw から `makeNormalMap()` 関数を用いて法線マップを生成し、テクスチャとして割り当てます（テクスチャユニット0）。

3. **環境マップ（キューブマップ）の設定 (`init()` 関数)**
   - キューブマップの6面分の画像ファイル (room2nx.raw 〜 room2pz.raw) を読み込み、`GL_TEXTURE_CUBE_MAP` にテクスチャ画像として割り当てます（テクスチャユニット1）。

4. **接線ベクトルの算出と設定 (sphere.cpp)**
   - 球を描画する際に、各頂点の法線ベクトルから接線ベクトルを算出します。
   - `glVertexAttrib3dv(tangentLoc, tangent)` を呼び出すことで、算出した接線ベクトルを attribute 変数としてバーテックスシェーダに渡します。

### 4.2 バーテックスシェーダ (embm.vert)

バーテックスシェーダでは、頂点単位で視点座標系における光線ベクトル `light`、視線ベクトル `view`、および接空間の基底ベクトル `t`, `b`, `n` を計算し、フラグメントシェーダに送ります。

1. **光線・視線ベクトルの計算**
   - 視点座標系の頂点位置 `position` から、光源へのベクトル `light` と、視点へのベクトル `view` を計算します。

2. **接空間の基底ベクトルの計算**
   - 視点座標系の法線ベクトル $\mathbf{n}$ (`normal`) を計算します。
   - アプリケーションから渡された接線ベクトル `tangent` を視点座標系に変換して正規化し、$\mathbf{t}$ とします。
   - 法線ベクトルと接線ベクトルの外積から、従接線ベクトル（Binormal / Bitangent）$\mathbf{b}$ を求めます（`b = cross(n, t)`）。
   - これら `t`, `b`, `n` は varying 変数としてフラグメントシェーダに送られます。

### 4.3 フラグメントシェーダ (embm.frag)

フラグメントシェーダでは、法線マップから取得した法線ベクトルを視点座標系に変換し、ライティングと環境マッピング（キューブマップのサンプリング）を行います。

1. **視点座標系への変換行列の作成**
   - バーテックスシェーダから受け取った接空間の基底ベクトル `t`, `b`, `n`（線形補間されたもの）を正規化し、接空間から視点座標系への変換行列 `toView = mat3(normalize(t), normalize(b), normalize(n))` を作成します。

2. **法線マップからの法線ベクトルの取得と変換**
   - `texture2DProj()` を用いて法線マップから色（RGB）を取得し、`fcolor.rgb * 2.0 - 1.0` によって接空間の法線ベクトルに復元します。
   - これに変換行列 `toView` を掛けることで、視点座標系における法線ベクトル `fnormal` を得ます。

3. **環境マップのサンプリングと最終色の決定**
   - 視点座標系の法線ベクトル `fnormal` と光線ベクトル `flight` を用いて拡散反射率 `diffuse = max(dot(fnormal, flight), 0.0)` を計算します。
   - 視線の反射ベクトル `reflect(fnormal, view)` を用いて、キューブマップテクスチャ `environment` を `textureCube` でサンプリングし、環境の映り込み成分 `specular` を得ます。
   - 最終的に得られた反射率と光源の色を用いて最終的なフラグメントの色 `gl_FragColor` を出力します。
